Report App / 実績管理 / Metrics V2

本番運用マニュアル

2026年6月時点の現行運用を前提にした、日次運用と本番投入のための実務マニュアルです。

1. 使う画面

実績管理アプリ

  • ログイン: /performance/login/
  • 日目標入力
  • 決済入力
  • 実績ダッシュボード確認
  • 過去実績確認

Metrics V2 / Report App

  • Metrics V2: 月 / 路程 / 期間での分析
  • ランキング / 属性分析
  • Report App: 全体報告の作成 / 修正

2. メンバーの日次運用

  1. 実績管理アプリにログイン
  2. 個人の日目標を入力
  3. 決済を登録
  4. 必要に応じて獲得メールを送信
  5. 活動終了を実行
  6. 実績管理 / Metrics V2 / Report App への反映を確認

UN

  • 通常の件数 / 金額ベースで登録
  • UN 用フォームが表示される

WV

  • CS / 難民 / CS+難民 で登録
  • CS は 1口 4,500円
  • 難民支援は金額変動
  • 件数は CS件数 と 難民件数 を分けて扱う

3. 管理者の日次運用

  1. 全体実績を確認
  2. 本日の活動状況を確認
  3. 有効メンバー一覧から個別状況を確認
  4. 必要なら補正実績を入力
  5. Metrics V2 で分析
  6. Report App で全体報告を確認
  7. 日報メールテンプレートを確認

補正実績の扱い

  • 補正実績は別枠で保持する
  • 最終的な累計実績には加算される
  • 個人実績にも加算される
  • 一覧・履歴では通常実績と分けて確認する

4. メール設定

設定場所: 管理者ページ → メール

  • Gmail 連携設定: 送信元メールアドレス / Client ID / Client Secret / Refresh Token
  • 決済報告用メールグループ設定: UN 用 / WV 用
  • 送信時は To = 自分, Cc = メールグループ
  • 異常時は 送信失敗 バッジと error message を確認して再送

5. 6月1日の本番投入手順

デプロイ前確認

  • Billing が有効
  • GitHub Actions の deploy workflow が有効
  • Cloud Run の Secret / env が正しい
  • Gmail 連携設定が保存済み
  • UN / WV のメールグループ設定が保存済み

本番投入直後の確認

  1. 実績管理ログイン
  2. UN の目標入力 / 決済登録
  3. WV の目標入力 / 決済登録
  4. Gmail 実送信
  5. 活動終了
  6. 実績管理 / Metrics V2 / Report App への反映確認
  7. 日報メールテンプレート確認

6. トラブル時の切り分け

画面が 500

  1. Billing
  2. Cloud Run revision log
  3. migrate job 成功有無
  4. Secret / env

メールが送れない

  1. Gmail 連携設定
  2. メールグループ設定
  3. 送信失敗 バッジ
  4. error message
