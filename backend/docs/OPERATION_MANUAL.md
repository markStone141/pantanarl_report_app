# 本番運用マニュアル

このドキュメントは、2026年6月時点の現行運用を前提にした実務向けマニュアルです。

対象
- メンバー
- 管理者
- 報告担当

---

## 1. 使う画面

### 実績管理アプリ
- ログイン: `/performance/login/`
- 主な用途
  - 日目標入力
  - 決済入力
  - 実績ダッシュボード確認
  - 過去実績確認

### Metrics V2
- 主な用途
  - 月 / 路程 / 期間での分析
  - ランキング
  - 属性分析

### Report App
- 主な用途
  - 全体報告の作成
  - 送信済み報告の確認 / 修正

---

## 2. メンバーの日次運用

### 基本フロー
1. 実績管理アプリにログイン
2. 個人の日目標を入力
3. 決済を登録
4. 必要に応じて獲得メールを送信
5. 活動終了を実行
6. 実績管理 / Metrics V2 / Report App へ反映を確認

### UN の決済登録
- 通常の件数 / 金額ベースで登録
- フォームは UN 用表示になる

### WV の決済登録
- `CS`
- `難民`
- `CS+難民`
のいずれかで登録

WV の注意点
- CS は 1口 4,500円
- 難民支援は金額変動
- 件数は `CS件数` と `難民件数` を分けて扱う

### 活動終了後
- 個人の実績ダッシュボードで当日の反映を確認
- 必要なら `過去実績` から日次実績 / 決済明細を修正

---

## 3. 管理者の日次運用

### 基本フロー
1. 実績管理アプリにログイン
2. `全体実績` を確認
3. `本日の活動状況` を確認
4. 有効メンバー一覧から個別状況を確認
5. 必要なら補正実績を入力
6. Metrics V2 で分析
7. Report App で全体報告を確認
8. 日報メールテンプレートを確認

### 補正実績の扱い
- 補正実績は別枠で保持する
- ただし最終的な累計実績には加算される
- 個人実績にも加算される
- 一覧・履歴では通常実績と分けて確認する

### WV の補正実績
- `CS`
- `難民`
- `CS+難民`
で入力
- `現場名` も記録する

---

## 4. メール設定

設定場所
- 管理者ページ → `メール`

事前設定
- Gmail 連携設定
  - 送信元メールアドレス
  - Client ID
  - Client Secret
  - Refresh Token
- 決済報告用メールグループ設定
  - UN 用
  - WV 用

送信時のルール
- 基本はメールグループ送信
- `To`: 自分
- `Cc`: 設定済みメールグループ

異常時
- `送信失敗` バッジが出る
- 復旧後に再送する

---

## 5. Report App の使い方

### ログイン
- 既に同一ブラウザでログイン済みなら、再入力なしで入れる場合がある

### できること
- 当日報告の作成
- 過去報告の確認
- 送信済み内容の修正

### 注意点
- 実績管理側の当日実績が初期値として反映される前提で使う
- WV は `CS件数` と `難民件数` の整合を確認する

## 6. 6月1日の本番投入手順

### デプロイ前確認
- Billing が有効
- GitHub Actions の deploy workflow が有効
- Cloud Run の Secret / env が正しい
- Gmail 連携設定が保存済み
- UN / WV のメールグループ設定が保存済み

### デプロイ
1. `main` に push
2. GitHub Actions で deploy 完了を確認
3. migration job 実行成功を確認

### 投入直後の確認
1. 実績管理ログイン
2. UN の目標入力
3. UN の決済登録
4. WV の目標入力
5. WV の決済登録
6. Gmail 実送信
7. 活動終了
8. 実績管理へ反映確認
9. Metrics V2 へ反映確認
10. Report App へ反映確認
11. 日報メールテンプレート確認

---

## 7. トラブル時の切り分け

### 画面が 500 になる
優先確認
1. Billing
2. Cloud Run revision log
3. migrate job 成功有無
4. Secret / env

### データが反映されない
1. 活動終了まで実施したか
2. 対象部署が UN / WV で混ざっていないか
3. 補正実績として入れていないか
4. 実績管理 / Report App の対象日を確認

### メールが送れない
1. Gmail 連携設定
2. メールグループ設定
3. `送信失敗` バッジ
4. error message の内容確認
