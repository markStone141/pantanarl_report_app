# 格言ライブラリ アプリ設計メモ

## コンセプト

「格言を読むアプリ」ではなく、格言を通して自分自身の考え方や価値観の変化を記録する個人向けナレッジアプリとする。

毎日の活動に入る前に「今日携えていく言葉」を選び、活動経験を通して同じ言葉への理解が変化していくことを蓄積する。

## 基本方針

- 認証は既存のログイン情報を利用する。
- 管理者と一般ユーザーで権限を分ける。
- CSVインポート、書籍・格言データ管理は管理者のみ。
- お気に入り、タグ、メモ、閲覧履歴はユーザー単位で管理する。
- SNS要素はMVPでは実装しない。
- 他人のメモ、コメント、いいね、ランキングは入れない。

## MVP

1. 書籍管理
   - 書籍一覧
   - 書籍詳細

2. 格言閲覧

   - 大テーマ、中テーマ、小テーマで探す

   - キーワード検索

3. 読書体験
   - 書籍の順番どおりに格言を読む

   - 気になった格言をクリックして詳細を開く

4. 格言詳細
   - 格言本文
   - テーマ
   - ページ
   - お気に入り
   - ユーザー専用タグ
   - メモタイムライン
   - 新規メモ追加

5. 個人記録
   - お気に入り
   - ユーザー専用タグ
   - 複数メモ
   - 閲覧履歴
   - よく読んでいる格言

## UI方針

アプリには大きく2つの導線を持たせる。

### 読む導線

書籍を本のように順番に読める導線。

- 書籍を選ぶ
- 格言が表示順に並ぶ
- 気になった言葉をタップする
- 詳細ビューでメモやタグを残す

モバイルでは一覧と詳細を画面切り替えにする。PCでは一覧と詳細パネルを並べることも検討する。

### 探す導線

必要な言葉にたどり着くための導線。

- 大テーマから探す
- 中テーマから探す
- タグから探す
- お気に入りから探す
- メモ済みから探す
- キーワード検索

## ホーム画面案

## AJAX方針

完全なReact SPAではなく、Django認証を使ったまま、Django template + AJAX + JSON APIで「ほぼ1ページアプリ」として実装する。

メイン画面は1ページに集約する。

```text
/quotes/
```

この画面内で以下をAJAXで切り替える。

- ホーム
- 書籍一覧
- 大テーマ一覧
- 中テーマ一覧
- 小テーマ一覧
- 格言一覧
- 格言詳細
- お気に入り
- タグ一覧
- メモ一覧
- 検索結果

ブラウザの戻る・進むに対応する場合は `history.pushState` を使う。

## 読み込み方針

読む体験では、ページめくりや詳細表示のたびにDBアクセスしない構成を優先する。

書籍1冊が数百件程度であれば、書籍を開いたタイミングで必要な格言データを一括取得する。

```text
/quotes/books/<book_id>/reader-data/
```

初回取得データの例。

```json
{
  "book": {
    "id": 1,
    "title": "格言集"
  },
  "quotes": [
    {
      "id": 1,
      "display_order": 1,
      "quote_text": "...",
      "large_theme": "...",
      "middle_theme": "...",
      "small_theme": "...",
      "page": 12,
      "is_favorite": false,
      "has_notes": true,
      "tags": []
    }
  ],
  "last_position": 35
}
```

DBアクセスの考え方。

- 読む画面の初回表示: 1回
- 前へ・次へ: なし
- 一覧から詳細表示: 原則なし
- メモ本文取得: 詳細表示時に必要であれば1回
- お気に入り保存: POST 1回
- メモ保存: POST 1回
- タグ保存: POST 1回

メモ本文はユーザーごとに増えるため、初回取得ではメモ件数だけ持ち、詳細表示時に本文を取得する方針も検討する。

## データモデル案

### 共通データ

```text
QuoteBook
- id
- title
- author
- created_at
- updated_at
```

著者は固定想定だが、将来の拡張に備えて任意項目として残す。

```text
Quote
- id
- book
- large_theme
- middle_theme
- small_theme
- quote_text
- page
- display_order
- created_at
- updated_at
```

テーマはMVPでは文字列として保持する。検索・分類が複雑化した場合にテーマモデルへ分離する。

### ユーザー別データ

```text
QuoteFavorite
- id
- user
- quote
- created_at
```

```text
QuoteUserTag
- id
- user
- name
- created_at
- updated_at
```

```text
QuoteTagging
- id
- user
- quote
- tag
- created_at
```

```text
QuoteNote
- id
- user
- quote
- body
- created_at
- updated_at
```

```text
QuoteView
- id
- user
- quote
- viewed_at
```

閲覧履歴が増えすぎる場合は、履歴用テーブルと集計用テーブルを分けることを検討する。

```text
UserBookPosition
- id
- user
- book
- quote
- updated_at
```

「前回の続きから読む」に使う。

```text
UserDailyQuote
- id
- user
- quote
- selected_date
- note
- created_at
- updated_at
```

「今日携えていく言葉」に使う。MVP後半または次フェーズで検討する。

- 必要に応じて全体利用状況の確認

## React利用について

MVPではDjango template + AJAXで実装する。

Reactを使う場合でも全体SPAではなく、格言リーダー画面だけをReact islandとして後から置き換える方針が安全。

React化候補。

- 格言リーダー
- タグ編集UI
- メモタイムライン
- 検索結果の即時更新
- 前後移動のアニメーション
